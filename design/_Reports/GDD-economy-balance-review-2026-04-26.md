## 2. 經濟模型概覽

### 2.1 收入來源（Faucets）

| 收入項目 | 金額範圍 | 觸發條件 |
|---|---|---|
| 委託預收（baseReward 全額入帳） | F=20g ～ SSS=3000g | 玩家接受委託 |
| 委託結算傭金（成功）| 基礎 20%；有會計時 22% | 任務成功 |
| condition 特質金錢獎勵 | `+baseReward × 0.5`（機率觸發）| 成功時條件特質 |

> 預收入帳後，`baseReward × 80%` 的金額在成功結算時實質退還（客戶回款），公會實得 20%。此三段節奏（預收→等待→淨結算）是核心金流體驗設計意圖。

### 2.2 支出來源（Sinks）

| 支出項目 | 金額範圍 | Jam 版狀態 |
|---|---|---|
| 委託結算賠償（失敗）| 基礎 10%；有會計時 8% | 完整實作 |
| 老手招募費 | D=100g / C=250g / B=500g / A=1000g / S=2000g | 完整實作 |
| 手動刷新招募板 | 150g / 次（每日 1 次免費）| 完整實作 |
| 建築升級費 | 150g ～ 40,000g（職員休息室 L5）| 完整實作 |
| 設施維護費 | FT-07 `OnGuildMaintenanceDue` | **Phase 2 no-op** |
| 職員薪水 | FT-08 `OnStaffSalaryDue` | **Phase 2 no-op** |

### 2.3 金流守恆關係

對於一筆 baseReward = R 的委託：

- **成功淨入帳** = R × effectiveCommissionRate ≈ `+0.20R`（默認）
- **失敗淨支出** = -R × effectivePenaltyRate ≈ `-0.10R`（默認）
- **成功/失敗金流比值** = 2:1（正面報酬是負面損失的 2 倍）

在 Jam 版無維護費/薪水 no-op 的情況下，**唯一穩定支出為建築升級費與招募費**；系統整體偏向淨收入正向，破產壓力主要來自頻繁失敗而非持續性支出。

## 3. 數值一致性審查

### 3.1 聲望變化表：雙版本衝突（HIGH）

**問題描述**：

`game-concept.md` 保留的 Phase 1 聲望變化表與 FT-04 GDD §3.6 `ReputationDeltaTable` 數值不同：

| 難度 | game-concept（Phase 1）成功/失敗 | FT-04 GDD（當前）成功/失敗 |
|---|---|---|
| F | +3 / -3 | +1 / -1 |
| E | +3 / -3 | +1 / -2 |
| D | +4 / -3 | +2 / -3 |
| C | +5 / -4 | +3 / -4 |
| B | +5 / -3 | +4 / -6 |
| A | +10 / -1 | +6 / -8 |
| S | +13 / -2 | +8 / -10 |

**差異分析**：
- F/E 難度：game-concept 成功 +3，FT-04 成功 +1（差距 3 倍）

- FT-04 版本明確標示為「Game Jam 初始資料（建議值）」，game-concept 標示為 Phase 1 legacy

**當前權威**：FT-04 GDD §3.6 的 `ReputationDeltaTable`

**風險**：若程式碼實作者或 CSV 填表者誤用 game-concept 版本，聲望收益將快速膨脹（F 難度每次成功 +3 vs +1），公會升等時間壓縮約 2/3，破壞 GuildLevelTable 的進度節奏。

### 3.3 死亡率異常：SS < S（MEDIUM）

**問題描述**：

來自 `【C-01】mission-database.md` / `game-concept.md` 的基礎死亡率：

| 難度 | baseDeathRate |
|---|---|
| A | 40% |
| S | 45% |
| SS | 40% |
| SSS | 65% |

SS 難度死亡率（40%）低於 S 難度（45%），打破了死亡率隨難度單調遞增的直覺預期。

**可能設計意圖**：SSS 作為「終局」任務的特殊設計，S 作為「天花板」單階難度，SS 為 S→SSS 之間的過渡，故死亡率設計不走線性；或 A=S=SS 皆視為「精英階」，共享相近死亡風險帶。

**風險**：若無明確設計意圖說明，玩家可能感到 SS 比 S「更安全」，導致 S 任務成為被刻意回避的難度。同時，成功時死亡率乘以 0.5 後，S=22.5%、SS=20%，差距微小，但仍有不一致感。

### 3.4 護送任務與成功率：特殊互動（MEDIUM）

### 3.5 NPC 意願門檻 vs. rankDiff=0 成功率（LOW）

**問題描述**：

FT-03 NPC 意願公式：
```
willingnessScore = finalSuccessRate - finalDeathRate × 0.5
effectiveScore + jitter(±0.10) < 0.25 → 拒絕
```

當 `rankDiff = 0`，基礎成功率 55%，假設無職業/種族/特質修正，且使用 B 難度（deathRate=18%）：
```
willingnessScore = 0.55 - 0.18 × 0.5 = 0.55 - 0.09 = 0.46
```
0.46 明顯高於拒絕閾值 0.25（含 jitter 最低 0.36），NPC **不會**拒絕。

但當 `rankDiff = -2`（冒險者比任務難度低 2 階），成功率 20%，B 難度：
```
willingnessScore = 0.20 - 0.18 × 0.5 = 0.20 - 0.09 = 0.11
```
加上最大 jitter +0.10 = 0.21，仍**低於** 0.25，NPC 必拒絕。

**評估**：此設計邏輯自洽——rankDiff ≤ -2 時高難度任務 NPC 正確拒絕。但需注意：`rankDiff = -1`（成功率 35%）+ A 難度（deathRate=40%）：
```
willingnessScore = 0.35 - 0.40 × 0.5 = 0.15
```
jitter 最大為 0.25，接近閾值邊界，NPC 行為呈隨機拒絕。此為設計意圖（玩家無法強迫 NPC 接必死任務），數值一致。

### 3.6 破產初始閾值與初始金幣的零緩衝（MEDIUM）

**問題描述**：

- `GOLD_INITIAL = 100g`（F-01 SystemConstants）
- E 階 `maxDebt = -100g`（C-06 WorldDangerTable）
- D 階招募費 = 100g（C-02 RecruitCostTable）

玩家起手金幣 100g，若立即招募 D 階冒險者（費用 100g），金幣將降至 0g，恰好觸及警告區間下限（因 `BankruptcyWarningState.Warning` 在 `threshold ≤ gold < 0` 時觸發，0g 在 Warning 之外，仍為 Normal 狀態）。

然而，若玩家招募 D 階後接了一張低難度任務**失敗**（例如 E 難度失敗賠償 -5g），金幣降至 -5g，進入 Warning 區間，破產倒數 3 小時（保險櫃 L1）立即啟動。

**分析**：對於 Jam 版，這是刻意設計的「起步壓力」——玩家應先接 F/E 任務累積資金後再招募 D 階冒險者，而非立即花光起手資金。但若教程/引導未說明此風險，玩家極可能在開局前幾分鐘就進入破產倒數，體驗落差較大。

### 3.7 世界危險度 E→D 破產閾值跳躍（LOW）

**問題描述**：

| 升階 | maxDebt 跳躍 | 倍數 |
|---|---|---|
| E → D | -100 → -500 | 5× |
| D → C | -500 → -1000 | 2× |
| C → B | -1000 → -2500 | 2.5× |
| B → A | -2500 → -5000 | 2× |

E→D 的跳躍幅度（5 倍）遠高於後續每階（2~2.5 倍），設計上反映「世界從和平狀態進入動盪是一個質變而非量變」。此為可接受的設計決策，但需確認：**破產倒數秒數（保險櫃等級）是否跟上 maxDebt 擴寬的節奏**。

保險櫃 L1=3h（10,800s），若玩家在 E→D 升階後金幣剛好在 -100~-500 範圍，3 小時的緩衝時間是否足夠完成一張中等難度任務回收金幣？

以 D 難度任務時長（150 分鐘 = 9,000s）為基準，3h = 10,800s > 9,000s，**勉強足夠接一張 D 難度任務來回**。但若玩家仍在接 E 難度（45 分鐘），3h 可以完成 4 張，緩衝空間尚可。

**風險等級**：低，但建議在 FT-07 §7 調整建議中說明「保險櫃 L1 的 3h 設定需配合 D 難度任務 2.5h 時長留有餘量」。

## 4. 經濟平衡分析

### 4.1 早期（E 階）經濟流

**初始狀態**：100g，無冒險者，委託板 L1（5 槽），公會 Lv1（可接 D 難度以下）

**建議起步路徑**（基於數值計算）：

1. 接 F 難度（baseReward=20g，預收入帳 +20g）
2. 派 F 階冒險者（招募免費）
3. F 難度成功傭金淨得 = 20 × 0.20 = +4g

**問題**：F 難度每次成功只得 4g，要從 100g 積累到 D 階招募費（再次需 100g 補充）需要大量重複。若玩家頻繁接 F 任務而非盡早切換 D/C 難度，成長極為緩慢。

**E 難度成功**：50g × 20% = +10g，更快速，建議引導玩家盡早接 E 難度。

### 4.2 中期（D~C 階）經濟流

**典型中期狀態**：持有 3~4 名 C 階冒險者，世界危險度 D/C，委託板 L2，金幣 500~2,000g

**典型收益計算（B 難度，baseReward=600g）**：

- 預收：+600g
- 成功結算：+120g（20% 傭金），淨得 120g
- 失敗結算：-60g（10% 賠償），淨損 60g

以 rankDiff=0（C 階冒險者接 C 難度任務）成功率 55%，期望值：
```
EV = 0.55 × 120 + 0.45 × (-60) = 66 - 27 = +39g
```

**期望值為正**，但方差大：連續 3 次失敗 = -180g，可能觸發破產警告。
建議玩家維持 `rankDiff ≥ +1`（70% 成功率），期望值 = 0.70 × 120 + 0.30 × (-60) = 84 - 18 = +66g。

**中期建築投資回收分析**：

委託板 L1→L2 費用 150g，增加 3 個委託槽。若每槽日均完成 2 張 D 難度（150g × 20% = 30g 淨得），3 槽/天 = 90g，回收周期 ≈ 1.7 天。投資效益合理。

### 4.3 後期（A 階）經濟流

**典型後期狀態**：S 階冒險者、A/S 難度任務、世界危險度 A

**S 難度成功（baseReward=1200g）**：
- 傭金（20%）= +240g
- 有會計加成（22%）= +264g
- condition `on_success_gold_bonus`（50% 觸發概率 × +600g bonus）= 期望 +300g

**S 難度期望收益**（A 階冒險者 rankDiff=+1，成功率 70%）：
```
EV = 0.70 × 240 + 0.30 × (-1200 × 0.10) = 168 - 36 = +132g（不含 condition）
```

**A 難度死亡率問題**：A 難度 baseDeathRate=40%，成功後折扣 0.5 = 20%。若派高階冒險者死亡，需花費 A=1000g 或 S=2000g 補充新成員，可能引發「高難度死人→缺人→掉難度→收益下降」的惡性循環。這是設計意圖中的「真實後果」，但數值上需確保玩家有足夠的聲望門檻來控制招募補充速度（FT-06 Lv4 才能招募 A 階）。

### 4.4 建築升級費用斜線分析

職員休息室升級費用呈非常陡峭的指數增長：

| 等級 | 升級費用 | 累計投入 | 效果（面試自動刷新縮短） |
|---|---|---|---|
| L1 | 500g | 500g | 24h 解鎖職員系統 |
| L2 | 1,500g | 2,000g | 18h |
| L3 | 4,500g | 6,500g | 12h |
| L4 | 13,500g | 20,000g | 8h |
| L5 | 40,000g | 60,000g | 6h |

L4→L5 需 40,000g，以 S 難度每筆淨得 240g 計算，需約 **167 次 S 難度成功**才能回收單次投資。此等級實際上是「傳奇公會」等級才能觸及的奢侈升級，Jam 版可能永遠達不到。

**評估**：作為長期目標設計合理，但 L3→L4（13,500g）的跳躍可能已超出 Jam 遊戲時長內的可及範圍，建議確認 Jam 版遊戲時長預設是否以 L3 作為終點目標。

### 4.5 Jam 版單向淨收入問題（維護費/薪水 no-op）

**問題描述**：Jam 版維護費與薪水管線均為 no-op，系統支出僅剩：

1. 一次性建築升級費（資金池型消耗，非定期扣款）
2. 一次性招募費（依需求觸發，非定期）
3. 任務失敗賠償（機率性，非穩定）

此設計導致**遊戲中不存在持續性負向壓力**：玩家即使建好職員休息室、招募了高稀有度職員，也不需要「維持」公會的定期費用。這讓遊戲感覺更接近「收集/成長」而非「管理/維持」，破產威脅主要來自集中性任務失敗而非慢性失血。

**對玩家幻想的影響**：「管理一個有真實後果的公會」的緊張感主要依賴任務失敗風險支撐，Jam 版下整體壓力偏低，玩家掌控感強但挑戰度略顯不足。

## 5. 玩家幻想契合度

### 5.1 目標情緒（MDA Aesthetics）

GDD 明確定義的目標 MDA Aesthetics：
- **Challenge**：任務失敗風險、死亡後果、破產倒數
- **Expression**：冒險者配置策略、公會升級路徑選擇
- **Fellowship**（二次）：冒險者 NPC 的個性化、命運感
- **Sensation**：結算視窗動畫、金幣跳動、破產紅字

### 5.2 掌控感評估

**正面因素**：
- 成功率公式透明（rankDiff 基礎 + 職業/種族/特質修正），玩家可預估風險
- 傭金率/賠償率數值固定且簡單（20%/10%），心算可得
- NPC 意願機制提供「冒險者也有自主性」的敘事感，同時保護玩家不派入必死任務

**負面因素**：
- F/E 難度每筆 +4g/+10g 回報極低，早期感覺「空洞感進度很慢」
- 公會升 Lv2 需聲望 100，以 B 難度 `+4/次` 計算需成功 25 次；以 F 難度 `+1/次` 計算需成功 100 次。起始難度（F/E）的聲望產出太低，早期進度感弱
- 死亡使冒險者永久消失，對投入大量培育感情的玩家打擊重，但 Jam 版冒險者個性化程度有限，情感連結不夠深時死亡會讓玩家感到「只是在耗費資源」而非「痛失夥伴」

### 5.3 成長感評估

**正面因素**：
- 公會等級升級帶來明確的質變（可接難度上限提升）
- 建築升級帶來容量擴充（委託槽、名冊容量）
- 世界危險度的升階可視為「關卡推進感」

**潛在問題**：
- 公會 Lv5 需聲望 1500，以 B 難度 `+4/次` 計算需 375 次 B 難度成功。若每張 B 任務時長 480 分鐘（8h），不考慮離線加速，需約 3,000 小時現實時間——顯然超出 Jam 版設計預期。這說明 Lv5 聲望門檻 1500 在 Jam 遊玩時長（假設 1~3 小時）內**幾乎無法達到**，Lv5 可能永遠是彩蛋性質的目標。

**建議確認**：Jam 版的「預期到達公會等級」是 Lv3 還是 Lv4？根據 GuildLevelTable：
- Lv2（聲望 100）：以 D 難度（+2/次）需成功 50 次，時間合理
- Lv3（聲望 300）：需 150 次 D 難度成功，或 75 次 C 難度成功，在 Jam 時長內仍可達
- Lv4（聲望 700）：需 175 次 C 難度，或 ~115 次 B 難度，邊界情況

### 5.4 張力與緊張感評估

**有效的緊張感設計**：
- 預收機制讓玩家「有錢但不敢花」，在等待結算期間形成心理張力
- 破產倒數（FT-07 保險櫃從 3h 到 48h）製造真實時間壓力
- 死亡率折扣（成功時 0.5）鼓勵派遣但不消除風險

**不足的緊張感設計**：
- Jam 版無維護費/薪水，缺乏「每隔一段時間必須確保有收入」的定期壓力
- 招募池 4 人候選/自動刷新 24h，玩家有充裕時間選擇，決策壓力較低
- 老手池 D=40%/C=30% 佔 70%，高階冒險者（A/S=12%）稀少，難度爬升需等待較長時間
